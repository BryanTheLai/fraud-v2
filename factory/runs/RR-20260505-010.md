# Run Record: Full Profile Observability Hardening

## Assignment

Tighten the Docker full profile so it proves functional API behavior and basic
observability, not just container startup.

## What Changed

- Added Grafana Prometheus data source provisioning.
- Added a Fraud V2 overview dashboard with decision volume, decision latency,
  event ingestion, and API target health panels.
- Expanded `scripts/full-smoke.ps1` to:
  - build and start the full profile
  - check API, Prometheus, Grafana, and Neo4j readiness
  - generate synthetic events through the protected API
  - score `user_00000` and assert a RED high-risk result
  - assert review queue and analyst dashboard content
  - assert fraud metrics are emitted
  - assert the Grafana dashboard loads
  - wait for Prometheus to scrape the API target
- Added tests that validate the Grafana dashboard, data source, and full smoke
  coverage.
- Added a GitHub Actions Docker image smoke that starts the API container,
  generates data, scores `user_00000`, and checks metrics.
- Made full-profile host ports configurable and gave `full-smoke.ps1` high
  default ports so it does not hit a separately running dev API on `8000`.
- Made `full-smoke.ps1` use a separate Docker Compose project name so it does
  not disturb a manually started full stack.
- Bumped package/API version to `0.10.0`.

## Issue Found During Verification

The first expanded smoke hit an existing local `uvicorn` process on
`127.0.0.1:8000` instead of the Docker API. The fix was to parameterize Compose
host ports and make the smoke use isolated host ports by default.

The second expanded smoke hit a PowerShell JSON edge case where
`Invoke-RestMethod` unwraps a one-item array. The fix was to wrap review case
results with `@(...)` before counting.

The third expanded smoke hit a Windows PowerShell HTML parser issue on local
dashboard HTML. The fix was to use `-UseBasicParsing` for smoke content checks.
