# Run Record: Tamper Evident Audit

## Assignment

Add local tamper-evident audit records for fraud events, decisions, review
workflow actions, and outbox state changes.

## What Changed

- Added `AuditEntry` and `AuditVerificationReport` domain models.
- Added `audit_entries` SQLite table with sequence, previous hash, payload hash,
  and entry hash.
- Added audit writes for:
  - event ingestion
  - decision creation
  - review case creation
  - review decision submission
  - outbox publish/failure state changes
- Added `SQLiteStore.verify_audit_chain()`.
- Added admin-only API endpoints:
  - `GET /v1/audit/entries`
  - `GET /v1/audit/verify`
- Expanded full smoke to verify audit entries and hash-chain validity.
- Added tests for valid chain verification and tamper detection.
- Updated docs and PR draft.
- Bumped package/API version to `0.13.0`.
