# Run Record: Add Local JWT Auth Boundary

## Assignment

Add an OIDC-shaped local JWT auth mode without requiring an external identity
provider or breaking the default dev-token path.

## What Changed

- Added `FRAUD_AUTH_MODE=jwt` settings.
- Added HS256 JWT validation with a required 32+ byte local secret and required
  `exp`, `iss`, `aud`, `sub`, and role claims.
- Added `fraud-v2 auth-token` for minting local test JWTs.
- Kept static local bearer tokens as the default auth mode.
- Added tests for valid JWTs, bad audiences, missing secrets, and role
  enforcement.
- Updated docs and PR draft.
- Bumped package/API version to `0.18.0`.
