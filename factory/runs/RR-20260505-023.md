# Run Record: Add Redpanda Stream Consumer

## Assignment

Add a local-safe Redpanda stream consumer that can ingest canonical event
envelopes into SQLite or Postgres and prove it in the full Docker profile.

## What Changed

- Added a `StreamIngestionWorker` with a small Kafka-compatible consumer
  protocol.
- Added a `fraud-v2 stream-consume` CLI command.
- Added store support for disabling outbox enqueue when an event arrived from
  the stream, preventing publish-consume-publish loops.
- Added unit coverage for successful ingestion, duplicate no-ops,
  idempotency-key conflicts, and invalid messages.
- Expanded full-profile smoke to publish a Redpanda event, consume it into
  Postgres, and verify the exact idempotency key is present.
- Updated docs, PR draft, and version metadata to `0.23.0`.
