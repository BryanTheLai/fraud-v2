# Run Record: Local RBAC

## Assignment

Add a local role-aware API authorization layer that moves the repo closer to a
production-shaped fraud system without adding external identity dependencies.

## What Changed

- Added `fraud_v2.security.auth` with:
  - `AuthRole.ADMIN`
  - `AuthRole.ANALYST`
  - `AuthRole.SYSTEM`
  - `AuthPrincipal`
  - role-token parsing and authorization helpers
- Added `FRAUD_API_TOKENS` support for comma- or semicolon-separated
  `role:token` bindings.
- Kept legacy `FRAUD_API_TOKEN` as an all-roles local dev token.
- Added `/v1/auth/whoami`.
- Restricted API surfaces by role:
  - `system` or `admin`: ingest, synthetic generation, scoring
  - `analyst` or `admin`: graph and review queue
  - all three roles: decision reads
- Added unit and integration tests for role boundaries.
- Updated setup, README, PR draft, and production-readiness docs.
- Bumped package/API version to `0.11.0`.
